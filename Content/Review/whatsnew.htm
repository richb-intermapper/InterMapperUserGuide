What's New in InterMapper®?

InterMapper includes many new features and enhancements. Here's a summary...

InterMapper Reports

InterMapper Reports provides feature, performance, and disk usage improvements, as well as additional built-in reports. See Using InterMapper Reports (Pg 1) (Pg 1) for more information.

  • Report template system - Version 5.5 introduces report templates, predefined reports that prompt the user for key settings, and use those settings to generate a report. The result can be saved as another report. See Creating a Report (Pg 1) for more information.
  • New, Load, and Save Reports pages - the Load and Save pages are easier to use, and the New Report page simplifies the creation of a new report. See Managing Reports (Pg 1) for more information.
  • New Report types - templates are available for Service Level Agreement, Event History, Dataset chart, Top "N", Details, and Outages.
  • CSV Export - InterMapper Reports now exports reports in CSV format. See Exporting and Linking to a Report (Pg 1) for more information.
  • Maintenance Tasks - improvements to the daily and weekly automatic maintenance tasks. Given enough free disk space, the maintenance tasks use a new behavior that takes less time, uses less memory, and improves the performance of subsequent database operations.
  • Performance and disk usage - The database now uses less disk space, adds data more quickly, and starts up faster.
    Notes:
    After the upgrade, the database initially consumes more space, but after the first weekly maintenance, its size is decreased substantially. You will see space savings only after the first weekly maintenance is complete.
    To achieve these improvements, raw data points are held in a new "datastore" table. If you have written SQL queries against the "datapoint" table, you need to modify your queries to use a new "load_data" function.
  • Quick report view - In the Map or List view, select a device, then choose in Reports from the Show submenu, available from the Monitor menu and the context menu, to view the Event History for that device.

Charts and InterMapper Database

InterMapper gives you greater visibility into and control over chart data, as well as data exported to InterMapper Database.

  • Easier access to Retention Policies The Retention Policies pane (in the System Preferences section of the Server Settings window) replaces the Retention Policies Tab of InterMapper DataCenter's web interface. Use it to add, delete, and edit retention policy definitions. For more information, see Retention Policies pane (Pg 1).
  • New Retention Policy parameters - Retention policies now include two new Server storage type parameters:
    • Forever - all charted or exported values are saved to a local disk file.
    • None - poll for the data but don't save it for charting or exporting.
      For more information, see Retention Policies pane (Pg 1).
  • Chart View - In Chart View, datasets are now displayed along with the charts containing them. View the chart's creation date, source, type, and filename for the associated chart file. Show the chart or device associated with a dataset, rename or remove the dataset, and export or delete data from the dataset.
    Note: Deleting data from one or more datasets may be time-consuming, and no polling activity takes place until the deletion is complete.
  • Status Window - In the Status Window,click a chartable value. A menu pops up, allowing you to choose to create a new chart or to raise and/or go to an existing chart which contains this dataset.
  • Dataset View - In the Device List window's new Dataset view, devices and interfaces with chartable values appear in the left pane of the window. The chartable values for the selected devices and interfaces appear in the right pane, with type and retention policy for each dataset. Set or adjust retention policies for the values from this view.

Improved Control of Device Labels

JavaScript in device labels and notifiers: Use Javascript to generate text in device labels, notifier titles, and notifier messages. Use JavaScript to access probe parameters and variables, as well as certain internal information, and use it in label text. This allows, for example, display of variables returned from a probe in a device label. See Dynamic Label and Alert Text (Pg 1) for more information.

Layer 2 Enhancements

  • IP Conflicts - InterMapper automatically indicates IP conflicts when it finds devices with the same IP address in more than one place.
  • Ignore IP Addresses - Use the Set Behavior… command (available from the Monitor menu or the context menu), and select the Ignore this address in Layer 2 box to indicate that a particular device’s IP address should be ignored in Layer 2.
  • In the Map or List view of the Map window, select a device, then choose in Layer 2 from the Show submenu, available from the Monitor menu and the context menu, to view that device's connections in the Device List Window's Layer 2 View.
  • Additional Tags - InterMapper automatically indicates the spanning-tree root and displays a loop icon where a switch port is plugged into another port on the same switch. In addition, the user can tag a switch port or VLAN manually as wireless and an endpoint as a virtual machine. See Using the Layer 2 View (Pg 1) for more information.
  • Minor tweaks to the server settings dialog - The server settings Layer 2 Features panel now includes a count of the number of Layer-2-enabled maps. It also contains a link to a dialog that allows you to enable or disable Layer 2 on a specific map. See Layer 2 Features (Pg 1) for more information.

Minor Features

  • New location of InterMapper Settings folder for Windows users - InterMapper's installer places the InterMapper Settings folder in Windows' preferred location unless you specify otherwise. For more information, see Files and Folders (Pg 1).
    Note: When you upgrade to , the installer checks for a Settings folder in the preferred location. If none is found, it attempts to move the existing Settings folder to the new location. If it fails, the upgraded version runs with the Settings folder in the old location.
  • Greater flexibility in handling licenses and serial numbers - A number of minor enhancements make it easier to enter your license or serial number into InterMapper.
  • Numerous bug fixes and minor improvements
