What's New in Version ?

Major Features

InterMapper Reports

This version represents continued performance, disk usage, and feature improvements to InterMapper Reports, as well as additional built-in reports.

  • Report templates - 5.5 introduces report templates, predefined reports that prompt the user for key settings, and use those settings to generate a report. The result can be saved as a report.
  • New, Load, and Save Reports pages - the Load and Save pages are easier to use, and the New Report page simplifies the creation of a new report.
  • New Report templates- templates are available for service level agreement, event history, dataset chart, top “N”, details, and outages.
  • Performance and disk usage - The way data is stored in the database has been redesigned. As a result, the database uses less disk space, adds data more quickly, and starts up faster. After the upgrade, the database initially consumes more space, but after the first weekly maintenance, its size is decreased substantially. You will see space savings only after the first weekly maintenance is complete. To achieve these improvements, raw data points are held in a new "datastore" table. If you have written SQL queries against the "datapoint" table, you need to modify your queries to use a new "load_data" function.
  • CSV Export - InterMapper Reports allows export of reports in CSV format.
  • Maintenance Tasks - We have made a number of improvements to the daily and weekly automatic maintenance tasks. Given enough free disk space, the maintenance tasks use a new behavior that takes less time, uses less memory, and improves the performance of subsequent database operations.

Charts and InterMapper Database

InterMapper gives you greater visibility into and control over chart data, as well as data exported to InterMapper Database.

  • Server Settings - The Reports Server panel now allows you to add, delete, and edit retention policy definitions directly, rather than requiring you to go to the IMDC interface. Note, too, that retention policies now include two “Server storage type” parameters along with the traditional database parameters: “Forever” and “None”. “Forever” means all charted or exported values are saved to a local disk file. “None” means that we will poll the data but not save it for charting or exporting.
  • Chart Window - In the Chart Window, you may now right-click on a dataset legend and choose “Show Device” to have InterMapper indicate in the Map or List Views which device the dataset belongs to.
  • Status Window - In the Status Window, when you click on a chartable value, a menu pops up, allowing you to choose to create a new chart or to raise and/or go to an existing chart which contains this dataset.
  • Chart View - In Chart View, datasets are now displayed along with the charts containing them. Creation date, source, type, and the filename for the associated chart file are also displayed. Users may show the chart or device associated with a dataset, rename or remove the dataset and export or delete data from the dataset.
    Note: Deleting data from one or more datasets may be time-consuming, and no polling activity takes place until the deletion is complete.
  • Dataset View - In the new dataset view, device and interfaces with chartable values are displayed in a filter on the left-hand side of the window. Select one or more devices or interfaces; the union of the chartable values appears on the right-hand side, along with the type and retention policy for the values. Set or adjust retention policies for the values here.

Improved Control of Device Labels

JavaScript in device labels and notifiers: Use Javascript to generate text in device labels, notifier titles, and notifier messages. Use JavaScript to access probe parameters and variables, as well as certain internal information, and use it in label text. This allows, for example, display of variables returned from a probe in a device label. See Generating Label Text with JavaScript for more information.

Minor Features

Layer 2 Enhancements

  • IP Conflicts - InterMapper will automatically indicate IP conflicts when it finds devices with same IP address in more than one place.
  • Ignore IP Addresses - Using “Set Behavior…”, the user can indicate that a particular device’s IP address should be ignored in Layer 2 by checking “Ignore this address in Layer 2”.
  • Additional Flags - InterMapper will automatically flag the spanning-tree root and ports where a switch port is plugged into another port on the same switch. In addition, the user may manually flag switch ports or vlans as wireless and endpoints as virtual machines by right-clicking on the switch port, vlan, or endpoint and selecting an item from the Tag submenu.
  • Minor tweaks to the server settings dialog - The server settings Layer 2 Features panel now includes a count of the number of Layer-2-enabled maps and a link to a dialog allowing you to enable or disable Layer 2 on specific maps.
  • New location of InterMapper Settings folder for Windows users - InterMapper's installer places the InterMapper Settings folder in Windows' preferred location unless you specify otherwise.

    Notes:
    When you upgrade to , the installer checks for a Settings folder in the preferred location. If none is found, it attempts to move the existing Settings folder to the new location. If it fails, the upgraded version runs with the Settings folder in the old location.

Other Enhancements

  • Greater flexibility in handling licenses and serial numbers - We have made a number of minor tweaks to make it even easier to enter your license or serial number into InterMapper.
  • Numerous bug fixes and minor improvements
