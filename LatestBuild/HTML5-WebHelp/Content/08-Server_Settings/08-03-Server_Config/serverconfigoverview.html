Server Configuration Panels

InterMapper provides three built-in servers you can use to view and retrieve information about the status of the network from remote computers. Each server's built-in firewall must be configured before it can be used. By default, each server's firewall is set up so that access is denied.

Use the Server Configuration panels of the Server Settings window to view and edit settings for the built-in servers, to manage users and groups, to control map access, and to manage a list of notifiers/alerts.

In the left pane of the Server Settings window, click the button for the settings you want to edit, as follows:

  • Remote Server - Start, stop, or edit settings for the InterMapper RemoteAccess server.
  • Reports Server - Start stop, or edit settings for the InterMapper Reports Server.
  • Web Server - Start, stop, or edit settings for the InterMapper web server.
  • Telnet Server - Start, stop, or edit settings for the InterMapper Telnet server.
  • Layer 2 Features - Turn on Layer 2 discovery for your server.
  • Enabled Maps - View a list of available maps, enable or disable maps, and import or export maps.
  • Users - View, Add, and Edit users and groups, and control access for users and groups.
  • Map Access - Control access by any user or group to any map through the web server or remote server.
  • Notifier List - View, add, copy, edit and remove notifiers.
  • SSL Certificate - Create new Certificate Signing Requests (CSR) and upload new certificates to the InterMapper server.

For more information on configuring your servers, see Server Access Control. It tells how to set a server's port, discusses encryption and when to use it, and describes how to configure a server's built-in firewall's list of IP addresses.
